# satmodel 平台界面介绍

## 1. 平台定位

`satmodel` 正在从一个以 Python 脚本为主的卫星姿态控制仿真工程，演进为一个可复现实验平台。

当前平台层已经具备下面这条完整工作流：

`场景配置 -> 实验计划 -> 运行实验 -> 结果归档 -> 图形化浏览`

它的重点不是替代底层动力学和控制模型，而是把这些能力组织成一个更适合展示、复查、批量实验和后续扩展的工作区系统。

## 2. 当前界面能做什么

本地中文控制台当前提供以下能力：

1. 自动扫描 `scenarios/` 下的场景和实验计划。
2. 提供左侧导航式平台工作区，按“平台总览 / 实验库 / 创建实验 / 计划管理 / 结果总览 / 结果对比 / 姿态回放 / Dashboard 预览”切换，避免所有内容堆成一条长页面。
3. 查看场景摘要，包括仿真时长、步长、控制器、环境和输出根目录。
4. 校验场景配置和实验计划配置。
5. 从已有场景快速创建新的实验计划。
6. 在实验设计页按“研究问题 -> 场景与变量 -> 任务与验收 -> 预览与生成”的分步工作台组织实验名称、变量模板、取值模板、任务模板、任务模式和输出目录。
7. 在“创建实验”里先按实验主线选择模板，并可在高级配置中启用第二扫描变量，生成二维权衡实验。
8. 提供“推荐实验库”，可直接打开或复用控制器整定、控制器 benchmark、轨道环境控制器 benchmark、环境敏感性、环境扰动分解、扰动-执行器权衡、测量质量敏感性、严格验收门限、Monte Carlo 鲁棒性、太阳指向切换、故障增益权衡、执行器能力边界和轮速管理实验。
9. 在创建实验时填写实验说明，并选择验收模板或自定义阈值，让实验计划从一开始就带有明确目标和通过标准。
10. 在实验计划列表或编辑器里把现有计划直接复制为副本，快速派生参数变体。
11. 在编辑器或最近计划区域里直接重命名实验计划，整理工作区中的实验版本。
12. 将旧计划归档到 `scenarios/archive/`，让活动工作区保持干净，同时保留历史文件。
13. 在“已归档计划”区域恢复历史计划，重新放回活动实验列表。
14. 直接点击“一键示例运行”，自动载入推荐场景、模板并创建运行完整演示。
15. 在“结果导览”中按推荐顺序跳转到 Run 排行、姿态回放和对应计划。
16. 直接运行实验计划，生成标准实验目录。
17. 浏览结果摘要，包括 run 数、通过率、最佳 run 和关键文件。
18. 在“当前结果”中自动生成“实验结论导读”，把研究问题、实验类型、当前观察和下一步建议压缩成一页摘要。
19. 在结果页显示“实验链路导航”，把当前结果放回标准实验主线里，说明当前所处环节以及推荐下一组实验。
20. 在结果页使用“概要 / 诊断 / 链路 / 产物”标签化摘要，减少长条页面滚动，更接近学术平台的结果阅读方式。
21. 在结果页新增“图表”标签，按实验类型推荐关键图、阅读重点和跳转入口，更接近学术实验报告的阅读节奏。
22. 在结果页汇总验收失败原因、角速度峰值、执行力矩峰值、扰动力矩峰值、主导扰动项、任务时间线和运行时快照摘要。
23. 自动生成“最差 Run 解释”，把失败原因、关键参数差异和边界工况线索压缩成一段可读说明。
24. 对比关键 run 的误差和控制力矩曲线。
25. 在页面内预览 `dashboard.html` 结果界面；静态 dashboard 本身也会展示验收失败原因、动态峰值摘要、环境扰动分解、最差 run 解释和任务/调度概览。
26. 进行本地三维姿态回放，并联动显示任务模式时间线和运行时调度。
27. 在总览页直接打开 `docs/` 中的路线、架构、实验库和参考资料。

## 3. 界面结构说明

平台控制台现在改成左侧导航式布局，右侧内容区按工作流切换：

- 平台总览：
  - 平台介绍
  - 实验介绍
  - 平台总览
  - 资料与扩展入口
  - 实验内容建议
- 实验库：
  - 实验资产地图
  - 一键示例运行
  - 推荐实验库
- 创建实验：
  - 实验设计工作台
  - 研究问题
  - 场景与变量
  - 任务与验收
  - 预览与生成
- 计划管理：
  - 实验池
  - 当前计划
  - 版本与归档
  - 场景与记录
  - 实验选择器与计划状态摘要
  - 批量校验与归档入口
  - 实验计划编辑器
  - 当前场景摘要与操作记录联动
- 结果总览：
  - 结果浏览
  - 当前结果摘要
  - 运行状态
- 结果对比：
  - 关键 run 对比
- 姿态回放：
  - 三维姿态回放
- Dashboard 预览：
  - `dashboard.html` 内嵌预览

这样更接近真正的平台界面：左侧常驻导航，右侧按任务切换，不需要在一长页里来回滚动。

## 4. 平台界面的核心展示点

### 4.1 场景和实验计划

场景是单次仿真的基础描述，负责说明：

- 时间参数
- 系统构造
- 控制器
- 环境
- 初始状态
- 输出设置

实验计划则在场景之上增加：

- 参数扫描
- Monte Carlo
- runtime 描述
- mission 描述
- 输出根目录
- 验收规则

这让平台不再依赖手写 Python 脚本组织实验，而可以直接用 JSON 驱动。

当前实验设计页还把实验主题进一步收敛为四类优先方向：

- 控制器整定
- 随机鲁棒性
- 任务模式切换
- 执行器能力边界

对应建议已整理到 [实验库建议](EXPERIMENT_LIBRARY.md)。

当前“创建实验”工作台也按成熟平台的实验设计顺序组织：

1. 先明确研究问题和实验名称。
2. 再选场景和实验模板。
3. 然后定义扫描变量、推荐取值和 Monte Carlo 规模。
4. 再补 mission、mode、参考目标和验收门限。
5. 最后统一确认输出目录、run 数和计划说明，再创建或直接运行。

### 4.2 结果摘要和关键 run 对比

每次运行实验后，平台会生成：

- `experiment_manifest.json`
- `study_manifest.json`
- `index.json`
- `summary_metrics.csv`
- `README.md`
- `dashboard.html`

界面会自动读取这些产物，并在“当前结果”区域汇总：

- 实验名称
- 所属场景
- run 数量
- 通过率
- 最佳 run
- 最佳末端误差
- 参数列和指标列

“关键 Run 对比”区域则可以对比关键 run 的：

- 末端姿态误差
- RMS 误差
- 峰值力矩
- 验收状态
- 姿态误差时序
- 执行力矩时序

### 4.3 三维姿态回放

平台界面支持基于 `time_history.csv` 中的真实四元数进行本地三维姿态回放。

当前回放具备：

- 播放/暂停
- 回到起点
- 时间滑条
- 当前姿态误差显示
- 当前模式显示
- 当前 task 显示
- 任务模式时间线联动
- 运行时调度模块联动

这部分特别适合演示“仿真不是只有表格和 CSV，而是有可理解的动态过程”。

## 5. 推荐展示流程

如果你要向老师、同事或项目成员演示，建议按这个顺序：

1. 打开平台首页，说明这是本地中文控制台。
2. 在“场景”中选择一个基础场景，展示场景摘要。
3. 在“创建实验”中基于该场景生成一个实验计划。
4. 在“实验计划”中校验并运行该计划。
5. 运行完成后，切到“当前结果”介绍实验摘要。
6. 打开“关键 Run 对比”，展示不同 run 的误差和力矩差异。
7. 展示“三维姿态回放”，说明模式切换和姿态变化是联动可见的。
8. 最后在“结果预览”中展示 `dashboard.html`，说明该结果目录可以脱离控制台单独打开和归档。

## 6. 与成熟平台范式的对应关系

当前设计不是随意堆功能，而是按成熟卫星仿真平台的范式组织：

- 借鉴 Basilisk：
  - process / task / module 的运行时分层
- 借鉴 Tudat：
  - environment setup / propagation setup / output 分离
- 借鉴 GMAT：
  - project / resource / mission sequence 的组织思路

在 `satmodel` 中，对应为：

- `PlatformProject`：工作区入口
- `ExperimentPlan`：实验计划契约
- `ExperimentRunner`：实验执行入口
- `ReportBuilder`：结果产物构建
- `dashboard.html`：静态结果浏览层
- `satmodel-platform-ui`：本地操作层

## 7. 当前版本的价值

当前版本已经不只是“能跑仿真”，而是已经具备平台雏形：

- 有统一入口
- 有配置驱动实验
- 有标准化结果目录
- 有中文图形界面
- 有动画和结果图
- 有运行时和任务模式的可视化联动

这意味着它已经适合用于：

- 课程展示
- 方案汇报
- 控制器对比实验
- 参数扫描复查
- Monte Carlo 结果整理
- 后续平台继续扩展

## 8. 当前边界

目前平台仍然是轻量版本，边界也很清楚：

1. 重点还在平台层，不在高保真物理重写。
2. runtime 和 mission 已可描述、可落盘、可展示，但还没有完全替换底层单速率执行器。
3. 三维姿态回放已经可展示，但仍是轻量本地回放，不是高保真三维任务场景。
4. 结果管理目前基于文件目录和 JSON/CSV/Markdown，还没有进入数据库和多用户协作阶段。

## 9. 启动方式

在仓库根目录运行：

```bash
python scripts/serve_platform_ui.py --root . --host 127.0.0.1 --port 8765
```

或者使用 CLI：

```bash
satmodel-platform-ui --root . --host 127.0.0.1 --port 8765
```

启动后在浏览器中打开：

`http://127.0.0.1:8765`

## 10. 下一步建议

下一阶段最值得继续补强的方向有三类：

1. 界面可操作性：
   - 继续细化分页导航
   - 增加实验模板卡和实验说明卡
   - 补更细粒度的结果筛选与搜索
2. 结果展示深度：
   - 更多诊断图
   - 轮速和扰动力矩预算图
   - 模式切换事件高亮
3. 平台工程化：
   - 工作区 manifest
   - 结果 schema 版本化
   - 更完整的 smoke test 与批量示例集
   - 面向 runtime、报告和 dashboard 的可插拔扩展位

这三块补齐后，平台会更接近一个可长期演进、可稳定展示、可团队复用的卫星姿态控制仿真系统。
